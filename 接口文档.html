启动流程

根据某个模型key，发起流程申请

请求：

xxxxxxxxxx
POST http://path:8082/workflow-api/instance/start
Content-Type: application/json
​
{
  "appSn":"xc_system",                 //系统标识（必填，标识流程引擎提供）
  "businessKey":"xc_20102103212312",   //业务单号（必填，类似于订单号）
  "applyUserCode":"01808724",          //发起人工号（必填）
  "operator":"02217992",               //当前操作人工号（必填，一般和发起人工号一致）
  "processDefinitionKey":"Process_1684747421812", //流程定义key(必填，流程引擎提供)
  "processInstanceName":"数据分发平台对接申请Process_1684747421812" //流程标题(非必填，如果不设置标题将为流程引擎生成),
   "variables":{ //流程变量(非必填，如果流程图中网关条件变量依赖流程变量则必须要要传递)
        "key1":"xxxx",
        "key2","xxxx",
        "key3":"xxxx",
        "key4":"xxxx"
    },
    "message":"备注信息"，//发起流程备注信息
   "customMark":"xxxx"  //自定义标记（非必填，给发起流程的设置一个标记，查询相关的接口可根据该标记查询）
}

返回：

xxxxxxxxxx
{
  "code": "SUCCESS_REQUEST",
  "success": true,
  "message": "success",
  "data": {
        "processInstanceId" : "372cd5e7-fe99-11ed-b608-005056be26d2" //流程实例id，可保存起来对流程引擎的操作（比如查询审批列表、流程图）
    }
}

流程引擎会默认将产生任务待办、执行抄送节点、流程完成时会向mq发送消息，不同的应用mq消息的tag不一样（tag为应用key），消息内容为字符串的json格式

主题名称（固定）：yto-workflow-notice-topic

通知类型

xxxxxxxxxx
    TASK_TO_DO("task_to_do", "任务待办通知"),
    TASK_TRANS("task_trans", "任务转办通知"),
    USER_ASSIGNEE("user_assignee", "审批人分配通知"),
    FLOW_READ("flow_read", "流程增加阅读者"),
    INST_DONE("inst_done", "实例完成通知"),
    BACK_START("back_start", "驳回通知"),
    RECALL("recall", "撤回通知"),
    OVER("over", "流程中止"),
    TASK_DONE("task_done", "已审批通知"),
